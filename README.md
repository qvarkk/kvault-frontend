# kvault-frontend

**Веб-интерфейс [kvault](https://gitverse.ru/qvarkk/kvault)** — самостоятельно хостируемой системы управления знаниями (заметки, веб-страницы, PDF-документы с автотегизацией и полнотекстовым поиском).

Это клиентское приложение работает поверх REST API kvault. Серверная часть — в репозитории [gitverse.ru/qvarkk/kvault](https://gitverse.ru/qvarkk/kvault).

---

## Возможности

| Функция | Описание |
| --- | --- |
| Заметки | Создание, редактирование и удаление заметок с поддержкой Markdown |
| Заметки по ссылке | Сохранение веб-страниц с превью, метаданными и извлечённым текстом |
| Теги | Ручная привязка тегов, переименование, фильтрация |
| Автотегирование | Подбор тегов по содержимому записи |
| Файлы | Загрузка PDF с индикатором прогресса и отменой |
| Онлайн-просмотр PDF | Открытие PDF прямо в браузере |
| Drag & Drop | Перетаскивание файлов для загрузки |
| Поиск и фильтры | Полнотекстовый поиск, фильтр по тегам, сортировка |
| Корзина | Восстановление и безвозвратное удаление |
| Темы | Светлая и тёмная |
| Локализация | Русский, английский, японский |
| Адаптивность | Корректное отображение на мобильных устройствах |

---

## Стек

Vue 3 (Composition API) · TypeScript · Vite · Tailwind CSS v4 · Pinia · Vue Router · shadcn-style компоненты на [reka-ui](https://reka-ui.com)

---

## Запуск для разработки

```bash
npm ci
cp .env.example .env   # указать VITE_API_URL
npm run dev
```

По умолчанию приложение ожидает API по адресу `http://localhost:6767/api/v1` (переменная `VITE_API_URL`). Для работы нужен запущенный бэкенд — см. [gitverse.ru/qvarkk/kvault](https://gitverse.ru/qvarkk/kvault).

Подробнее о командах и правилах по коду — в [CONTRIBUTING.md](./CONTRIBUTING.md).

## Развёртывание

В продакшене фронтенд собирается в Docker-образ и поднимается вместе со всем стеком kvault одной командой `docker compose`. Инструкция по развёртыванию (включая домен, реверс-прокси и HTTPS) — в репозитории бэкенда: **[HOSTING](https://gitverse.ru/qvarkk/kvault)**.
